# EduRAG 智能写作助手 — 项目开发进度

> 基于 RAG（检索增强生成）的本地化 K12 教育辅助系统，专注高考语文写作训练。
> 技术栈：Python + ChromaDB + BGE Embedding + Ollama (Qwen3:8B) + Flask + React + Vite + TypeScript + Ant Design + Recharts

---

## 项目结构

```
EduRAG/
├── core/                    # 核心引擎
│   ├── db_manager.py        # ChromaDB 数据库管理
│   ├── embedding.py         # BGE 中文向量嵌入
│   ├── llm_client.py        # Ollama LLM 客户端
│   ├── rag_pipeline.py      # RAG 查询流水线
│   ├── reranker.py          # BGE 重排序模型
│   └── retriever.py         # 向量检索 + 元数据过滤
├── api/                     # Flask API 服务
│   ├── app.py               # 应用入口 + CORS
│   └── routes/
│       ├── __init__.py      # 通用过滤条件构建
│       ├── search.py        # 知识库检索 3 接口
│       ├── writing.py       # 写作训练 4 接口
│       ├── practice.py      # 强化训练 7 接口
│       ├── portfolio.py     # 作品集管理 10 接口
│       └── upload.py        # 知识库上传 2 接口
├── subjects/                # 学科 Prompt 模板
│   └── chinese/
│       ├── prompt_loader.py # 模板渲染引擎
│       ├── writing.py       # 写作训练器封装
│       └── prompts/         # 4 套 Prompt 模板
├── scripts/                 # 数据导入工具
│   ├── doc_extractors.py    # PDF/DOCX 文本提取
│   ├── text_chunker.py      # 智能文本分块
│   ├── metadata_parser.py   # 文件名元数据解析
│   ├── import_docs.py       # 知识库批量导入
│   ├── exam_parser.py       # 试卷结构化解析
```

## 2026-06-18 更新记录

### 命题热点模块修复与优化

#### 已完成工作

**1. LLM响应处理错误修复** ✅
- **问题诊断**：发现`llm.chat()`返回的是字典对象（Ollama API格式），而代码假设是字符串并直接调用`.strip()`，导致`AttributeError: 'dict' object has no attribute 'strip'`错误
- **解决方案**：
  - 添加类型检查，判断response是字典还是字符串
  - 从字典中正确提取`message.content`字段
  - 支持多种响应格式（OpenAI兼容格式等）
  - 改进JSON解析前的内容提取逻辑
- **涉及文件**：
  - `api/routes/hot_topics.py` - 修改`_generate_hot_topics`函数和`generate_custom_prompt`函数

```python
if isinstance(response, dict):

    elif 'choices' in response and len(response['choices']) > 0:
        content = response['choices'][0].get('text', '').strip()
```
