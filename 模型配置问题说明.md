## 为什么需要重启？

### Python应用的配置加载机制

EduRAG后端在**启动时**执行以下流程：

1. **读取config.yaml** → 获取模型配置
2. **初始化OllamaClient** → 传入模型名称
3. **创建Flask应用** → 注册所有API路由
4. **开始监听请求** → 处理前端调用

```python
# api/app.py 第55-83行
config = load_config()  # ← 只在启动时读取一次
ollama_cfg = config.get('ollama', {})

llm = OllamaClient(
    model=ollama_cfg.get('model', 'qwen2.5:7b'),  # ← 启动时确定模型
    ...
)
```

**关键点**：配置只在启动时读取，运行时修改config.yaml不会自动生效！

## 重启步骤

### 方法1：手动重启（推荐）

1. **找到运行后端的终端窗口**
2. **按 `Ctrl+C`** 停止服务
3. **重新启动**：
   ```bash
   python start_backend.py
   ```

### 方法2：使用start.bat

双击 `start.bat` 会自动重启所有服务（Ollama + 后端 + 前端）

## 常见问题

### Q: 为什么修改代码后也要重启？
A: Python是解释型语言，但Flask应用在启动时会将所有代码加载到内存。修改`.py`文件后需要重启才能生效。

### Q: 前端修改需要重启吗？
A: 不需要！前端使用Vite热更新，保存文件后浏览器自动刷新。

### Q: 重启会丢失数据吗？
A: 不会！数据存储在ChromaDB和JSON文件中，重启不影响数据。

## 总结

| 项目 | 是否需要重启 | 说明 |
|------|-------------|------|
| 修改config.yaml | ✅ 需要 | 配置只在启动时读取 |
| 修改Python代码 | ✅ 需要 | 代码在启动时加载 |
| 修改前端代码 | ❌ 不需要 | Vite自动热更新 |
| 添加新数据 | ❌ 不需要 | 数据实时写入文件 |
