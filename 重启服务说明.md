# 重启服务说明

## 问题诊断

智能命题热点模块返回 **404 Not Found** 错误的原因：

**正在运行的后端服务是旧版本**，在我们添加 `hot-topics` 路由之前就已经启动了，所以不包含这个新的API端点。

## 解决方案

### 1. 重启后端服务（必需）

**重要：** 必须使用 `pixel_ai` conda环境启动后端，该环境包含GPU版本的PyTorch。

```bash
# 停止当前运行的后端服务（Ctrl+C 或关闭后端窗口）

# 使用 pixel_ai 环境启动后端
cd D:\PixelSmile\EduRAG
conda activate pixel_ai
python -m api.app
```

**为什么必须用 pixel_ai 环境？**
- 项目配置要求 GPU 版本的 PyTorch (`2.5.1+cu121`)
- 本地默认Python环境没有CUDA支持
- 使用默认环境会报错：`AssertionError: Torch not compiled with CUDA enabled`

### 2. 重启前端服务（推荐）

虽然前端代码已更新，但为了确保所有更改生效，建议也重启前端：

```bash
# 停止当前前端服务（Ctrl+C）

cd D:\PixelSmile\EduRAG\frontend
npm run dev
```

### 3. 验证修复

重启后，测试以下功能：

#### 测试后端API直接调用
```bash
# 测试分类接口
curl http://localhost:5000/hot-topics/categories

# 应该返回类似：
# {"success":true,"categories":[...]}
```

#### 测试前端页面
1. 访问 http://localhost:3000/hot-topics
2. 点击"搜索热点"按钮
3. 应该看到：
   - 加载提示："正在分析热点话题，请稍候..."
   - 成功时显示话题列表
   - 如果LLM不可用，会显示降级示例数据

### 4. 检查Ollama服务

确保Ollama正在运行且模型可用：

```bash
# 检查Ollama服务
ollama list

# 检查qwen3:8b模型是否存在
ollama ps

# 如果模型不存在，拉取它
ollama pull qwen3:8b
```

## 常见问题

### Q1: 启动后端时报错 "Torch not compiled with CUDA enabled"

**原因：** 没有使用 pixel_ai 环境

**解决：**
```bash
conda activate pixel_ai
python -m api.app
```

### Q2: Ollama服务未启动

**错误：** `Ollama 服务不可用，请确保已启动: ollama serve`

**解决：**
```bash
ollama serve
```

### Q3: 模型不存在

**错误：** `模型 qwen3:8b 不可用且无法自动拉取`

**解决：**
```bash
ollama pull qwen3:8b
```

### Q4: 仍然返回404

**可能原因：**
1. 后端没有完全重启
2. 端口冲突

**解决：**
```bash
# 查找占用5000端口的进程
netstat -ano | findstr :5000

# 终止所有占用进程（替换PID）
taskkill /F /PID <PID>

# 重新启动后端
conda activate pixel_ai
python -m api.app
```

## 服务状态检查清单

启动成功后，应该有：

- ✅ Ollama服务运行中（端口11434）
- ✅ 后端API运行中（端口5000）
- ✅ 前端开发服务器运行中（端口3000）
- ✅ 浏览器可以访问 http://localhost:3000

   - 不会显示空白页面
