## 💡 关于 Bun

### 什么是 Bun？
Bun 是一个现代的 JavaScript 运行时，比 Node.js 更快，支持 TypeScript 原生运行。

### 是否需要安装 Bun？
**不需要！** 我们已经提供了 Node.js 版本，功能完全相同。

### 如果想安装 Bun
Windows 用户可以在 PowerShell 中执行：

```powershell
irm bun.sh/install.ps1 | iex
```

然后就可以使用：
```bash
bun run export_conversation_bun.ts
```

## 🔧 常见问题

### Q1: 运行脚本时提示"文件不存在"
**解决**: 检查对话记录文件路径是否正确
- 默认路径: `C:\Users\deejo\.qoder-cn\cache\projects\EduRAG-8fad6596\conversation-history\task-327\task-327.jsonl`
- 如果 task 编号不同，请修改脚本中的路径

### Q2: 输出是乱码
**解决**: 确保使用 UTF-8 编码的终端
- Windows: 执行 `chcp 65001` 切换到 UTF-8
- 或使用 PowerShell 而非 CMD

### Q3: 没有权限写入 docs 目录
**解决**: 以管理员身份运行终端，或修改输出路径
