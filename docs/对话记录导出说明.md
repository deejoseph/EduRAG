# EduRAG 对话记录导出工具 - Bun 版本

## 📋 简介

这是一个使用 Bun 运行时导出的 Qoder 对话记录为 Markdown 格式的工具，专为写书用途设计。

## 🚀 使用方法

### 方法一：直接运行（推荐）

```bash
bun run export_conversation_bun.ts
```

### 方法二：使用 bunx

```bash
bunx tsx export_conversation_bun.ts
```

### 方法三：先编译再运行

```bash
# 如果已安装 TypeScript
tsc export_conversation_bun.ts --target es2020 --module commonjs
node export_conversation_bun.js
```

## 📁 输出文件

导出的 Markdown 文件将保存在：
```
docs/conversation_YYYY-MM-DD_HHMMSS.md
```

例如：`docs/conversation_2025-01-XX_143022.md`

## ✨ 功能特性

1. **完整的对话记录** - 包含所有用户和 AI 助手的消息
2. **结构化格式** - 使用 Markdown 标题、代码块等格式化输出
3. **工具调用记录** - 显示所有工具调用的名称和参数
4. **统计信息** - 包含用户消息数、AI 消息数等统计数据
5. **时间戳** - 自动添加导出时间和消息编号

## ⚙️ 配置说明

### 修改输入文件路径

如果需要导出其他对话记录，请修改 `export_conversation_bun.ts` 中的 `jsonlPath` 变量：

```typescript
const jsonlPath = 'C:\\Users\\deejo\\.qoder-cn\\cache\\projects\\EduRAG-8fad6596\\conversation-history\\task-XXX\\task-XXX.jsonl';
```

### 修改输出目录

默认输出到 `docs/` 目录，可以修改 `outputDir` 变量：

```typescript
const outputDir = 'exports'; // 改为 exports 目录
```

## 🔧 故障排查

### 问题 1: 找不到 Bun 命令

**解决方案**: 安装 Bun
```bash
# Windows (PowerShell)
irm bun.sh/install.ps1 | iex

# macOS/Linux
curl -fsSL https://bun.sh/install | bash
```

### 问题 2: 文件不存在错误

**错误信息**: `❌ 对话记录文件不存在: ...`

**解决方案**: 
1. 检查文件路径是否正确
2. 确认 Qoder 缓存目录位置
3. 查看实际的 task 目录编号

### 问题 3: 解析失败警告

**警告信息**: `⚠️  解析失败: ...`

**说明**: 这是正常的，JSONL 文件中可能包含空行或特殊格式的行，工具会自动跳过。

### 问题 4: 编码问题

如果出现乱码，可能是文件编码问题。当前工具默认使用 UTF-8 编码读取。
