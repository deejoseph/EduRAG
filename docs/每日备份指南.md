# EduRAG 每日备份脚本使用指南

## 📋 简介

`daily_backup.sh` 是专为 EduRAG 项目设计的自动化备份脚本，用于每天工作结束时快速备份数据和导出对话记录。

## ✨ 功能特性

1. **自动备份 data 目录** - 包含向量数据库和所有文档
2. **自动导出对话记录** - 保存当天的 AI 对话历史
3. **智能清理旧备份** - 保留最近 7 天的备份，最多保留 3 个数据备份
4. **带时间戳的备份命名** - 便于查找和管理

## 🚀 使用方法

### 在 Git Bash 中运行

```bash
cd /d/PixelSmile/EduRAG
./daily_backup.sh
```

### 或者使用 bash 命令

```bash
cd /d/PixelSmile/EduRAG
bash daily_backup.sh
```

## 📁 备份输出

备份文件将保存在 `backup/` 目录下：

```
backup/
├── data_20260623_180723/        # data 目录备份
│   ├── chroma_db/               # 向量数据库
│   └── docs/                    # 原始文档
├── conversation_2026-06-23_180723.md  # 对话记录
└── ... (其他历史备份)
```

## 🔄 备份策略

### 自动清理规则
- 删除超过 7 天的备份
- 最多保留最近 3 个数据备份
- 对话记录不会被自动清理

### 建议的执行时间
- 每天工作结束前
- 重要修改完成后
- 关闭计算机前

## 💡 最佳实践

### 1. 建立日常习惯
每天下班前执行一次备份：

```bash
# 完成当天工作后
cd /d/PixelSmile/EduRAG
./daily_backup.sh
```

### 2. 重要操作前后备份
在进行以下操作前后都执行备份：
- 大规模数据导入
- 数据库结构修改
- 系统升级或重构

### 3. 定期检查备份
每周检查一次 backup 目录，确保：
- 备份文件大小正常
- 备份时间戳正确
- 磁盘空间充足

## 🔧 自定义配置

如果需要修改备份策略，可以编辑 `daily_backup.sh` 文件：

### 修改保留天数
```bash
# 原配置：保留 7 天
find . -maxdepth 1 -type d -name "data_*" -mtime +7 -exec rm -rf {} \;

# 修改为：保留 14 天
find . -maxdepth 1 -type d -name "data_*" -mtime +14 -exec rm -rf {} \;
```

### 修改保留数量
```bash
# 原配置：保留最近 3 个
ls -dt data_* 2>/dev/null | tail -n +4 | xargs rm -rf

# 修改为：保留最近 5 个
ls -dt data_* 2>/dev/null | tail -n +6 | xargs rm -rf
```

## ⚠️ 注意事项

1. **首次备份可能较慢** - 如果 data 目录很大，第一次备份需要较长时间
2. **确保磁盘空间** - 每个备份大约几百 MB 到几 GB
3. **不要中断备份** - 等待脚本完全执行完毕
4. **Git Bash 必需** - 此脚本需要在 Git Bash 环境中运行

## 📊 备份验证

备份完成后，可以检查：

```bash
# 查看备份目录
ls -lh backup/

# 查看最新备份的大小
du -sh backup/data_*/

# 查看对话记录
ls -lh docs/conversation_*.md
```

## 🛠️ 故障排查

### 问题 1: "Permission denied"
**解决方案**: 
```bash
chmod +x daily_backup.sh
```

### 问题 2: "command not found"
**解决方案**: 确保使用 Git Bash 而非 PowerShell

### 问题 3: 备份失败
**解决方案**: 
- 检查磁盘空间
- 确保 data 目录存在
- 检查文件权限
