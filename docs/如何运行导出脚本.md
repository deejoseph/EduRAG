# 如何运行对话记录导出脚本

## 🎯 快速开始

### 方法一：在 Git Bash 中运行（推荐）

```bash
# 进入项目目录
cd /d/PixelSmile/EduRAG

# 运行 Node.js 版本（无需安装 Bun）
node export_conversation_node.js
```

### 方法二：在 PowerShell 中运行

```powershell
cd d:\PixelSmile\EduRAG
node export_conversation_node.js
```

### 方法三：在 CMD 中运行

```cmd
cd d:\PixelSmile\EduRAG
node export_conversation_node.js
```

---

## 💡 关于 Bun

如果您确实想使用 `bunx`，需要先安装 Bun：

### Windows 安装 Bun

**方法 1：使用 PowerShell（推荐）**
```powershell
irm bun.sh/install.ps1 | iex
```

**方法 2：使用 npm**
```bash
npm install -g bun
```

安装完成后，就可以使用：
```bash
bun run export_conversation_bun.ts
# 或
bunx tsx export_conversation_bun.ts
```

## 🔧 故障排查

### 问题 1: "找不到 node 命令"
**解决方案**: 
- Node.js 应该已经随系统安装
- 检查安装：`node --version`
- 如果未安装，从 https://nodejs.org 下载

### 问题 2: "文件不存在: C:\Users\deejo\.qoder-cn\..."
**解决方案**:
- 检查 Qoder 缓存目录是否存在
- 查看实际的 task 编号（可能不是 task-327）
- 修改脚本中的文件路径

### 问题 3: 权限错误
**解决方案**:
- 以管理员身份运行终端
- 或确保对 docs 目录有写入权限
