# EduRAG 启动说明

## 一键启动（推荐）

双击 `start.bat` 文件即可自动完成以下步骤：

1. ✅ 检查并启动 Ollama 服务
2. ✅ 预加载 Qwen3:8B 模型
3. ✅ 启动后端 API 服务（端口 5000）
4. ✅ 启动前端开发服务器（端口 3000）
5. ✅ 自动打开浏览器访问 http://localhost:3000

### 启动窗口说明

启动后会看到 **3 个窗口**：

| 窗口标题 | 说明 | 关闭影响 |
|---------|------|---------|
| EduRAG 智能写作助手 - 启动器 | 主启动脚本 | 关闭不影响服务 |
| EduRAG Backend | 后端 API 服务 | 关闭后 API 不可用 |
| EduRAG Frontend | 前端开发服务器 | 关闭后网页无法访问 |

## 手动启动（调试用）

如果一键启动失败，可以手动分步启动：

### 1. 启动 Ollama（如已运行可跳过）

```bash
ollama serve
```

### 2. 启动后端 API

**重要：** 必须使用 `pixel_ai` conda 环境，该环境包含正确配置的 PyTorch GPU 版本。

```bash
cd D:\PixelSmile\EduRAG
conda activate pixel_ai
python -m api.app
```

后端将在 http://localhost:5000 启动

**为什么需要 pixel_ai 环境？**
- pixel_ai 是项目的专用 conda 环境
- 包含 **GPU 版本的 PyTorch** (`2.5.1+cu121`，CUDA 12.1)
- 可充分利用 RTX 3070 8G 进行 GPU 加速
- Embedding 编码和重排序速度提升约 10 倍
- 避免与本地其他 Python 环境的依赖冲突

### 3. 启动前端

```bash
cd D:\PixelSmile\EduRAG\frontend
npm run dev
```

前端将在 http://localhost:3000 启动

## 常见问题

### 问题 1：后端启动超时

**错误提示：** `[!] 后端启动超时`

**可能原因：**
1. 依赖缺失
   ```bash
   pip install -r requirements.txt
   ```

2. 端口 5000 被占用
   ```bash
   # 查看占用进程
   netstat -ano | findstr :5000
   # 终止进程（替换 PID）
   taskkill /F /PID <PID>
   ```

3. 模型加载失败
   - 检查网络连接（需要从 HuggingFace 下载模型）
   - 首次启动可能需要 2-3 分钟

### 问题 2：前端启动失败

**可能原因：**
1. 依赖未安装
   ```bash
   cd frontend
   npm install
   ```

2. 端口 3000 被占用
   ```bash
   # 查看占用进程
   netstat -ano | findstr :3000
   # 终止进程
   taskkill /F /PID <PID>
   ```

### 问题 3：Ollama 服务未运行

**解决方法：**
```bash
# 启动 Ollama
ollama serve

# 拉取模型（如未下载）
ollama pull qwen3:8b
```

### 问题 4：PyTorch CUDA 错误

**错误提示：** `AssertionError: Torch not compiled with CUDA enabled`

**原因：** 未使用 pixel_ai 环境或环境中的 PyTorch 不是 GPU 版本

**验证 pixel_ai 环境的 PyTorch：**
```bash
# 直接使用 pixel_ai 环境的 Python
C:\Users\deejo\anaconda3\envs\pixel_ai\python.exe -c "import torch; print(torch.cuda.is_available())"

# 应输出: True
```

**如何创建 pixel_ai 环境（如不存在）：**
```bash
conda create -n pixel_ai python=3.10
conda activate pixel_ai
pip install -r requirements.txt
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
```

### 问题 5：conda 命令找不到

**错误提示：** `'conda' is not recognized...`

**解决方法：**
```bash
# 使用完整路径
C:\Users\deejo\anaconda3\Scripts\activate.bat pixel_ai
```

## 停止服务

要停止所有服务：

1. 关闭 **EduRAG Backend** 窗口（或按 Ctrl+C）
2. 关闭 **EduRAG Frontend** 窗口（或按 Ctrl+C）
3. （可选）关闭 Ollama：`taskkill /F /IM ollama.exe`

## 技术栈

- **后端**: Python 3.10+ / Flask / ChromaDB / BGE Embedding
- **前端**: React 18 / TypeScript / Ant Design / Vite
- **AI**: Ollama (Qwen3:8B) / BGE-reranker
- **数据库**: ChromaDB (嵌入式向量数据库)
- **PyTorch**: 2.5.1+cu121（GPU 版本，CUDA 12.1）

## GPU 加速配置

当前 pixel_ai 环境已配置 **GPU 版本的 PyTorch** (`2.5.1+cu121`)，可充分利用 RTX 3070 8G。

**验证 GPU 是否启用：**

启动后端后，在控制台应看到：
```
INFO:core.embedding:Embedding 模型加载成功: BAAI/bge-base-zh-v1.5 (device: cuda)
```

如果显示 `device: cpu`，请检查：
1. config.yaml 中 `embedding.device` 和 `reranker.device` 是否为 `"cuda"`
2. pixel_ai 环境的 PyTorch 是否为 GPU 版本：
   ```bash
   C:\Users\deejo\anaconda3\envs\pixel_ai\python.exe -c "import torch; print(torch.cuda.is_available())"
   # 应输出: True
   ```

**性能对比：**
| 操作 | CPU | GPU (RTX 3070) |
|------|-----|----------------|
| Embedding 编码 | ~500ms/句 | ~50ms/句 |
| 重排序 | ~200ms/次 | ~20ms/次 |
| 首次检索 | ~2s | ~0.5s |
